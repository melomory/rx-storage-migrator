# Миграция из Directum 5

## Назначение

Данный документ описывает особенности использования `RxStorageMigrator` при миграции данных из **Directum 5** с использованием **SSIS**.

## Общие требования

Для корректной работы мигратора необходимо:

- использовать **Microsoft SQL Server**;
- иметь доступ к базе данных исходной системы **Directum 5**;
- обеспечить доступ к таблицам исходной системы:
  - `MBAnalit`
  - `SBEDocVer`
  - `SBEDoc`
- предварительно создать служебные таблицы, используемые в процессе миграции.

## Подготовка базы данных

Перед запуском миграции в базе данных Directum 5 должны быть созданы служебные таблицы.

Готовый SQL-скрипт:

- [create-converter-tables.sql](./create-converter-tables.sql)

## Служебные таблицы

В процессе миграции используются следующие таблицы:

### `Converter_Documents_EDocVersion_Current`

Таблица хранения информации о текущих версиях электронных документов и сопоставления данных исходной и целевой систем.

Основные поля:

- `XRecID` — идентификатор документа в Directum 5
- `Body_Id` — идентификатор тела документа в целевой системе
- `Storage` — идентификатор хранилища
- `RXDocId` — идентификатор документа в целевой системе

### `Converter_DocVer_Transferred`

Таблица фиксации перенесённых версий документов.

Используется для:

- контроля уже обработанных версий;
- хранения статуса миграции;
- записи комментариев;
- хранения путей к файлам в исходной и целевой системах.

Основные поля:

- `XRecID` — идентификатор документа в Directum 5
- `Number` — номер версии
- `RXDocId` — идентификатор документа в целевой системе
- `BodyId` — идентификатор тела версии
- `MigrStatus` — статус миграции
- `Comment` — комментарий или описание ошибки
- `D5FilePath` — путь к файлу в Directum 5
- `RXFilePath` — путь к файлу в целевой системе

### `Converter_Documents_Sorted`

Таблица со списком документов, подготовленных к обработке в определённом порядке.

Основные поля:

- `XRecID` — идентификатор документа в Directum 5

### `Converter_Documents_StorageD5`

Таблица описания хранилищ исходной системы Directum 5.

Основные поля:

- `Id` — идентификатор хранилища
- `Name` — наименование хранилища
- `StorageType` — тип хранилища:
  - `F` — файловое хранилище
  - `S` — хранение в SQL Server
- `FSPath` — путь к файловому хранилищу для типа `F`

> Для хранилищ типа `S` поле `FSPath` не заполняется.

### `Converter_Documents_StorageRX`

Таблица описания хранилищ целевой системы.

Основные поля:

- `Id` — идентификатор хранилища
- `Name` — наименование хранилища
- `FSPath` — путь к файловому хранилищу целевой системы

## Требования к подключению

Для выполнения миграции должно быть настроено подключение к БД Directum 5, из которой доступны для чтения таблицы:

- `MBAnalit`
- `SBEDocVer`
- `SBEDoc`

Также учетная запись, от имени которой выполняется миграция, должна иметь права:

- на чтение данных исходной системы;
- на чтение и запись в служебные таблицы;
- на выполнение используемых SQL-запросов.

## Проверка перед запуском

Перед запуском миграции рекомендуется проверить, что:

- используется **MS SQL Server**;
- служебные таблицы созданы;
- таблицы `MBAnalit`, `SBEDocVer`, `SBEDoc` доступны;
- пути в таблицах хранилищ заполнены корректно;
- у пользователя есть необходимые права;
- подключения настроены корректно.

## Примечания

- Мигратор использует данные и служебные таблицы в БД **Directum 5**.
- Все SQL-запросы ориентированы на **Microsoft SQL Server**.
- При изменении структуры таблиц совместимость мигратора должна быть проверена отдельно.
