## 🚀 Сборка проекта

Проект использует **NUKE Build** через локальные `dotnet tools`.

### 1️⃣ Восстановление инструментов

Перед первой сборкой необходимо установить локальные инструменты:

```pwsh
dotnet tool restore
```

Это установит NUKE версии, указанной в `dotnet-tools.json`.

### 2️⃣ Сборка проекта

```bash
dotnet nuke compile
```

или явно указав target:

```bash
dotnet nuke --target Compile
```

### 3️⃣ Полная сборка с тестами

```bash
dotnet nuke --target Test
```

### 4️⃣ Альтернативный способ (через скрипты)

Можно использовать вспомогательные скрипты:

```bash
build.cmd
```

или

```pwsh
./build.ps1
```

⚠️ Рекомендуемый способ — через dotnet tool restore и dotnet nuke, так как это гарантирует использование версии NUKE, зафиксированной в репозитории.

## 🧪 Тестирование

- Все тесты находятся в папке `tests`
- Новая функциональность должна сопровождаться тестами
- PR без тестов может быть отклонён

Запуск тестов:

```csharp
dotnet test
```

## 🎨 Стиль кода

Проект использует:

- `.editorconfig`
- `stylecop.json`

Перед отправкой PR убедитесь, что:

- Нет предупреждений StyleCop
- Код отформатирован
- Нет неиспользуемых using

Рекомендуется включить автоформатирование в IDE.

## 🌿 Работа с ветками

Основные правила:

- Ветка main — стабильная
- Для изменений создавайте отдельную ветку:

  ```bash
  feat/имя-фичи
  fix/описание-багфикса
  refactor/описание
  ```

## ✅ Pull Request

Перед созданием PR:

- Обновите ветку от main
- Убедитесь, что сборка проходит
- Убедитесь, что тесты проходят
- Обновите CHANGELOG.md (если требуется)

PR должен содержать:

- Краткое описание изменений
- Обоснование изменений
- При необходимости — пример использования


## 📝 Версионирование

Проект использует GitVersion.
